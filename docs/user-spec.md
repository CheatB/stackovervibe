# Stackovervibe — Что мы делаем и зачем

## В одном предложении
Красивый сайт в стиле хакеров из 80-х, где любой вайбкодер найдёт структурированные знания, готовые инструменты и понятные гайды — вместо того чтобы собирать всё по кускам из YouTube, Telegram и ChatGPT.

## Для кого это

**Основная аудитория:** ребята, которые уже попробовали вайбкодить и обожглись.
- Тратят деньги на токены, а результат — мусор
- Тратят время на настройку процесса, а он не складывается
- Ищут ответы везде, но нигде нет системы

**Вторичная аудитория:** новички, которые только слышали про вайбкодинг и хотят начать правильно.

## Как это работает

### Сценарий 1: Новичок хочет начать вайбкодить
**Контекст:** человек посмотрел видео про вайбкодинг, загорелся, гуглит "как начать вайбкодить"
1. Находит Stackovervibe в поисковой выдаче
2. Попадает на страницу "Путь новичка" — пошаговый маршрут
3. Видит первый шаг: "Что такое вайбкодинг и с чего начать"
4. Идёт по шагам: установка → настройка → первый проект
5. На каждом шаге — конкретный гайд, расписанный "для дебилов" со всеми пояснениями
6. Где нужно — скачивает готовые конфиги одной кнопкой или копирует код
7. В итоге имеет настроенную среду и понимание процесса
**Если что-то пошло не так:** в каждом гайде есть раздел "Частые проблемы"

### Сценарий 2: Опытный вайбкодер ищет конкретный инструмент
**Контекст:** человек уже кодит с ИИ, но хочет улучшить процесс
1. Гуглит "хуки Claude Code настройка" или заходит напрямую
2. Попадает в каталог инструментов
3. Находит нужную категорию (хуки, скиллы, команды, правила)
4. Открывает инструмент — видит специфичные для типа поля (триггер, синтаксис, workflow)
5. Код: копирует / скачивает файлом / смотрит на GitHub
6. Применяет у себя
**Если что-то пошло не так:** инструкция по установке и решение типовых ошибок

### Сценарий 3: Пользователь хочет поделиться своей наработкой
**Контекст:** вайбкодер сделал крутой хук/скилл и хочет поделиться
1. Логинится через Telegram-бота (открывается новая вкладка с Telegram)
2. Система получает вебхук об авторизации
3. Пользователь нажимает "Написать пост"
4. Заполняет в rich-editor: заголовок, категория, описание, код, файлы
5. Может сохранить как черновик и вернуться позже
6. Отправляет на модерацию
7. Может редактировать пока пост на модерации
8. Модератор (Иван) одобряет или отклоняет в CMS
9. При отклонении — причина из списка + комментарий, уведомление через TG-бота
10. При одобрении — пост публикуется в общем каталоге, уведомление через TG-бота
11. После публикации редактирование невозможно

### Сценарий 4: Пользователь взаимодействует с контентом
**Контекст:** человек читает статью/инструмент
1. Ставит лайк/дизлайк (без авторизации)
2. Хочет прокомментировать → логинится через Telegram
3. Пишет комментарий → публикуется сразу (без модерации)
4. Хочет поделиться → кнопки "Telegram" или "Копировать ссылку"
5. Проваливается в профиль автора → видит имя, аватарку, список постов, рейтинг

## Что входит в первую версию (MVP)

### Контент и навигация
- **Главная страница** — стильный лендинг в ретро-хакерском стиле
- **Путь новичка** — жёсткий линейный флоу с пошаговыми гайдами
- **Каталог инструментов** — специфичные поля для каждого типа (скилл, хук, команда, правило)
- **Описание фреймворка** — общая картина
- **Полнотекстовый поиск** по всему контенту
- **CMS-панель** — rich-editor с блоками кода и кастомными компонентами

### Действия с кодом
- Копировать в буфер
- Скачать файлом
- Посмотреть на GitHub (если есть ссылка)

### UGC и социал
- **Авторизация** через Telegram-бота (новая вкладка + вебхуки)
- **Пользовательские посты** — rich-editor, черновики, модерация, публикация в общем каталоге
- **Комментарии** под статьями (авторизованные, без модерации)
- **Лайк/дизлайк** под каждой страницей (без авторизации)
- **Профиль пользователя** — имя и аватарка из Telegram, список постов, рейтинг
- **Рейтинг** — посты + лайки
- **Шаринг** — Telegram + копировать ссылку

### Модерация
- Только Иван через CMS
- Отклонение: выпадающий список причин + комментарий
- Уведомления через TG-бота (одобрен / отклонён с причиной)
- Бан пользователей, удаление комментариев

### Аналитика и SEO
- Yandex.Metrika + Google Analytics с первого дня
- Максимальная SEO-оптимизация каждой страницы

### Прочее
- Баннер cookies + страница политики конфиденциальности

## Что НЕ делаем сейчас

- **Английская версия** — потом, когда будет стабильный контент
- **Проверка подписки на TG-канал** — потом, когда будет канал
- **Видео-контент** — потом
- **Интерактивные примеры кода** — потом
- **Монетизация** — потом, сначала трафик
- **RSS** — не делаем
- **Ограничения на файлы** — потом если понадобятся

## О чём надо подумать (риски и нюансы)

- **Контент нужно создать** — фреймворк и инструменты есть, но оформленных статей пока нет
- **Фреймворк переписывается** — на сайте будет обновлённая версия, контент наполняется по мере готовности
- **SEO требует времени** — сайт не попадёт в топ на следующий день
- **Один модератор** — если поток UGC вырастет, нужны будут дополнительные модераторы
- **Качество UGC** — нужно следить чтобы пользовательский контент не снижал качество сайта
