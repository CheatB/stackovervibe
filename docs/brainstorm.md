# Stackovervibe — Brainstorming

## Пользователи и их мир

### Два сегмента:
1. **Приоритет:** Ребята, которые уже попробовали вайбкодить и обожглись
   - Тратят кучу токенов, а результат говно
   - Тратят кучу времени на настройку процесса, а он не строится
2. **Второй:** Новички, которые только слышали про вайбкодинг

### Где сейчас ищут ответы:
- YouTube, Telegram, Reddit, ChatGPT — всего понемногу
- **Нигде нет структуры** — информация размазана кусками

### Уровень технической грамотности:
- Разный, но объединяет одно — хотят кодить с ИИ без боли

## Продукт

### Суть:
Stackovervibe (Stack Overflow + Vibe) — структурированная база знаний по вайбкодингу.
Место, где есть ответы на все вопросы без лишнего шума от ИИ.

### Структура контента — гибрид:
- **Путь новичка** (линейный маршрут): от "что такое вайбкодинг" до продвинутых техник
- **Справочник** (по категориям): для опытных — зашёл, нашёл, забрал

### Контент на MVP:
- Описание фреймворка работы (общая картина)
- Отдельная страница на каждый инструмент (скиллы, хуки, команды, правила)
- Возможность скачать конфиги/шаблоны
- Только контент Ивана (UGC потом)

### Типы контента MVP:
- Гайды и статьи
- Каталог инструментов с описаниями
- Скачиваемые конфиги и шаблоны
- Видео — нет (потом)
- Интерактивные примеры — нет (потом)
- Q&A формат — нет (потом, с UGC)

## Дизайн

### Стиль:
- **Тёмная тема**
- **Ретро-хакерский вайб 80-х** — Hackerman из Kung Fury
- Зелёные терминалы, глитч-эффекты, ASCII-арт, неон, CRT-свечение
- Клише из фильмов про хакеров
- Сайт должен быть "на приколе" с визуальными отсылками
- **Красота и стильность — приоритет**

## Технические решения

### Язык:
- Русский (английская версия потом)

### CMS:
- Нужна CMS-панель с первой версии
- Сейчас для Ивана, потом для других авторов

### Инфраструктура:
- VPS — текущий сервер (с нуля: Nginx, Docker, SSL)
- Домен: stackovervibe.ru (куплен)
- Стек: на выбор архитектора

### SEO:
- Главный канал трафика
- Максимальная оптимизация под все запросы по вайбкодингу и смежные

## Бизнес

### Монетизация:
- Пока не думаем, потом
- Возможно: курсы, подписки, реклама, B2B (по модели Stack Overflow)

### UGC (пользовательский контент):
- В перспективе — другие пользователи выкладывают наработки
- Обязательная проверка на вредоносность
- Q&A формат (как Stack Overflow)

### Амбиция:
- Завируситься
- Поднять в поисковую выдачу
- Стать THE местом про вайбкодинг на русском

## Контент-процесс

### Создание:
- Иван + ИИ: накидывает суть → ИИ оформляет → доводит до нужного ToV
- Частота: по мере готовности

### Готовый контент:
- Есть фреймворк работы и примочки (конфиги, скиллы, хуки, команды, правила)
- Оформленного контента пока нет — будем создавать
